1. Consent

By using our Service, you hereby consent to our Privacy Policy and agree to its terms.

2. Information We Collect

We collect information primarily through third-party services and localized app functions to provide and improve our Service.

Types of Data Collected (via Third-Parties):

  • Device Identifiers and Usage Data: Unique device IDs, IP addresses, advertising identifiers (AdMob), and usage patterns.
  • Diagnostic Data (Firebase Crashlytics): Crash traces and system activity data to help us fix bugs.

3. Use of Sensitive Permissions & APIs

To provide automated clicking and color detection, our Service requires access to the following sensitive Android APIs and Frameworks:

3.1 AccessibilityService API

We use the AccessibilityService API to perform automated clicks, swipes, and multi-touch gestures. This is essential for the "Auto Clicker" functionality.

  • No Data Collection: We DO NOT use the Accessibility API to collect any personal or sensitive information.
  • No Setting Changes: We DO NOT change user settings or prevent users from disabling/uninstalling any service without permission.

3.2 Shizuku Framework (Seamless Mode)

Our Service integrates the Shizuku Framework to enable "Seamless Mode" for advanced automation tasks.

  • Purpose: Shizuku allows the app to execute shell-level commands for screen interaction and system automation with higher precision and lower latency.
  • Local Execution: All commands issued via Shizuku are executed locally on your device. We do not use Shizuku to access, modify, or upload any personal files or system settings unrelated to the app's automation features.
  • User Control: Users must manually authorize Dopifier via the Shizuku Manager app. You can revoke this permission at any time.

3.3 MediaProjection API (Screen Capture)

Our Service uses the MediaProjection API to capture screen content for real-time color identification.

  • Temporary Local Storage: Screenshots taken for "Recording" or "Inspect Color" features are stored locally in the App's internal cache/folder.
  • Ephemeral Processing: These images are processed locally on your device to detect colors and are never transmitted to any external servers.

4. How We Use Your Information

  • Provide, operate, and maintain the automation Service.
  • Analyze screen colors to trigger user-defined automated actions.
  • Improve app stability via crash reporting.
  • Serve advertisements via AdMob.

5. Data Storage and Deletion

Regarding the screen captures and action logs saved by the app:

  • Internal Storage: All captured images are saved only in the application's private directory (App Folder).
  • User Control: Users can manually delete any "Inspect Color" records and their associated image files within the app interface.
  • Full Erasure: Uninstalling the application will automatically delete all locally stored data, images, and configuration files created by the app.

6. Third-Party Service Providers

  • Google AdMob: For personalized and non-personalized advertising.
  • Google Firebase: For analytics and crash reporting.
  • Shizuku API: For system-level automation interface.
